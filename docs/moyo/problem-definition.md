## JTBD (Job To Be Done)

유튜브에서 찾은 레시피를 따라 요리할 때, 이전의 실패 경험과 나만의 변형 이력을 기억하기 위해, 같은 실수를 반복하지 않고 매번 더 나은 결과를 만들고 싶다.

## 페르소나

유튜브 레시피 영상으로 요리하는 본인.

## 핵심 페인 포인트 (우선순위순)

1. **실패 반복 / 누적 학습 부재** — 이전 실수를 매번 처음부터 떠올려야 함. 같은 실패를 반복하게 됨.
2. **회상 비용** — 이전에 봤던 영상·결과·문제 지점을 기억해내는 데 시간·에너지를 과도하게 소모.
3. **저품질 영상 재선택** — 결과가 좋지 않았던 영상을 또 선택하게 됨. 영상 자체에 대한 평가가 누적되지 않음.
4. **커스텀 변형 망각** — "지난번엔 간장을 절반만 넣었었는데" 같은 본인의 조정 이력이 사라짐. 나만의 버전이 만들어지지 않음.
5. **간 조절 실패** — 어디서 어떻게 조정했는지 메모가 없어 개선 불가.

## 빈도·심각도

- **빈도**: 동일 메뉴를 격주(2주에 1회)로 반복 시도. 시도할 때마다 발생.
- **심각도**: 높음. 시간 낭비(회상 비용) + 결과 품질 저하(실패 반복) + 동기 저하("또 실패할 것 같다") 세 가지가 복합 작용.

## 핵심 인사이트

- **검색은 문제가 아니다.** 유튜브에 레시피는 충분히 많고, 사용자가 검색을 못 해 포기한 적은 한 번도 없음.
- 진짜 문제는 **회상 + 누적 학습 부재**. 풍부한 supply에 비해 빈약한 personal memory가 갭의 본질.
- 사용자 스스로 "이건 도구 부재의 문제이지 내가 멍청한 게 아니다"로 인식 — 시스템 문제로 명확히 귀인하고 있음.
- 현재 우회 수단은 **0** (메모 앱·유튜브 좋아요·재생목록 등 어떤 것도 사용 안 함) → unmet need.

## 비-목표

- 레시피 자동 추천 / 새 레시피 탐색 (탐색은 문제가 아님 — 사용자 명시)
- 유튜브 외 플랫폼 (블로그, 앱 레시피 등)
- 식단·칼로리 관리, 건강 목적 기능
- 식재료 구매·재고 관리

## 솔루션 가설 (보류 — 참고용)

사용자가 제시한 가설:
- 유튜브 API 활용 → 메뉴명 검색 → 영상 리스트 표시
- 사용자가 따라한 영상 체크 + 메모
- 영상 description / 고정 댓글에서 레시피 텍스트 노출

→ 문제 정의는 합의됨. 솔루션 형태는 prd-writer 단계 이후에 논의.
